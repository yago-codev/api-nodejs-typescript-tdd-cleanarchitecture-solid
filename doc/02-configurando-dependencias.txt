01) Inicalizando o "package.json":

    yarn init -y


02) Padroninzando commits seguindo a convenção do site "conventional commits"

    link:
    https://www.conventionalcommits.org/en/v1.0.0/


03) Adicionando dependência de desenvolvimento para 
    fazer com que o Git obedeça a um padrão de commits pré-estabelecidos:

    O que o "git commit message linter faz?
    Ele irá monitorar nosso commit para 
    ver se ele está de acordo com as regra de padronização do "convenctional commits"...

    git add git-commit-msg-linter -D


04) Criando o ".gitignore":

    Inicialmente iremos ignorar a pasta
    "node_modules"


05) Adicionando o "typescript" e o pacote "@types/node" como dependencias de desenvolvimento:

    yarn add typescript @types/node


06) Criando o arquivo de configuração do TypeScript:

    raíz do projeto -> tsconfig.json


07) Estrutura de configuração do arquivo "tsconfig.json":

    {
      // opções de configuração da transpilação do TS
      "compilerOptions": {
        // diretório de saída onde o arquivo transpilado será gerado:
        "outDir": "./dist",
        // padrão/sintaxe de import/export utilizado no projeto:
        "module": "commonjs",
        // versão em que o JS será gerado quando for transpilado:
        "target": "es2019",
        // vai fazer com que TODOS módulos consigam utilizar o padrão "import/export"
        "esModuleInterop": true,
        // vai incluir qualquer tipo de arqui js dentro da pasta de saída, mesmo que sejam JSON's
        "allowJs": true
      },
    }
